# Anti-Slop Design Reference

Anti-slop improves execution quality. It does not override source-brand trust cues, category conventions, or product mechanism communication.

Use this after the preservation hierarchy is satisfied.

## What anti-slop is for

- removing obvious AI design tells
- improving rhythm, hierarchy, and interaction quality
- keeping pages from collapsing into generic premium-tech sludge

## What anti-slop is not for

- silently redesigning the brand
- replacing category-legit trust cues with trendier patterns
- forcing every page into the same editorial taste package

## Banned patterns

### Typography slop
- Inter as default body/headline choice unless the source already uses it
- giant H1 used to fake hierarchy
- generic system font stacks when the brand has a real type direction

### Color slop
- purple/blue AI gradients by default
- pure black `#000000`
- neon glows and untinted shadows
- accent-color spam

### Layout slop
- centered-everything hero by default
- three equal feature cards by default
- uniform section spacing everywhere
- card soup

### Content slop
- vague premium-tech language
- fake precision numbers
- stock trust-badge lookalikes
- generic testimonial placeholders presented as real

## Required patterns

### Hierarchy
- one dominant idea per section
- real above-fold CTA
- visible trust cue above fold
- proof placed where it reduces friction, not buried because it looks cleaner

### Typography
- intentional font pairing or faithful brand font usage
- controlled line lengths
- weight and spacing used for hierarchy

### Layout rhythm
- asymmetry where appropriate
- varied section pacing
- constrained content width
- clean mobile collapse

### Interactions
- buttons feel clickable
- focus states visible
- accordion behavior accessible and robust

## Brand fidelity guardrail

Before applying an anti-slop suggestion, ask:
1. Does this preserve trust-critical category conventions?
2. Does this preserve mechanism communication?
3. Does this fit the actual brand visual language?
4. Is this improvement real, or just imported taste?

If 1-3 are not yes, do not apply the change.

## Final anti-slop check

- [ ] hero does not look like a generic AI template
- [ ] anti-slop changes did not erase source trust cues
- [ ] mechanism remains visually and verbally legible
- [ ] there is no card-soup or equal-card defaulting
- [ ] buttons, spacing, and shadows feel intentional
- [ ] the page still looks like this brand and this category

If the page looks stylish but less trustworthy or less clear, anti-slop was misapplied.
